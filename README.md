# ハンドメイド イベント販売価格シミュレーター

ハンドメイドアクセサリー作家向けの、イベント販売に特化した価格計算WEBアプリです。  
原価・制作時間・イベント出店コストを入力すると、「最低価格」「推奨価格」「ブランド価格」を自動で算出します。

---

## アプリ概要

サッポロモノヴィレッジのようなハンドメイドイベントに向けて、
感覚ではなく数字で価格を決める手助けをします。

**算出する価格:**

| 価格 | 意味 |
|------|------|
| 最低価格 | 赤字にならない下限の価格 |
| 推奨価格 | 売れやすさと利益のバランスを取った価格 |
| ブランド価格 | 作品の世界観・ブランド感を守れる価格 |

**主な特長:**

- スマホで快適に使えるレスポンシブデザイン
- 入力内容の自動保存（ブラウザのlocalStorage）
- 心理価格帯（1500円・1800円・2500円など）への自動丸め
- イベント経費を全商品で按分する実務的な計算
- 警告表示（赤字リスク・価格が高すぎる場合など）

---

## 使い方

1. **商品の原価**：パーツ代・金具代・梱包資材費などを入力
2. **制作コスト**：制作時間（分）と作業単価を入力
3. **イベント出店コスト**：ブース料・交通費・駐車場代などを入力
4. **売上想定**：イベント全体の販売予定数（按分計算に使用）を入力
5. **利益設定**：最低限ほしい利益額と理想の利益率を設定
6. **ブランド設定**：価格帯感（控えめ/標準/高め）を選択
7. 右側の**結果エリア**に3種類の価格が表示されます

> 「サンプル入力」ボタンを押すと、実際の販売例をもとにした数値が自動入力されます。

## 計算ロジックの説明

### 1. 材料費合計
```
materialsCost = パーツ代 + 金具代 + 梱包資材費 + 台紙費 + その他材料費
```

### 2. 作業コスト
```
laborCost = 制作時間（分） / 60 × 作業単価（円/時間）
```

### 3. イベント経費の按分
```
eventCostPerItem = イベント経費合計 / 今回の販売予定数（全商品）
```
イベント経費（ブース料・交通費など）は全商品で割り勘して負担します。

### 4. 総原価（1個あたり）
```
totalCost = materialsCost + laborCost + eventCostPerItem
```

### 5. 最低価格
```
minimumPrice = totalCost + 最低限ほしい利益額
```

### 6. 推奨価格
```
candidate = totalCost / (1 - 理想利益率)
recommendedPrice = max(candidate, minimumPrice × 1.1)
```
最低価格を必ず上回るよう保証しています。

### 7. ブランド価格
```
brandPrice = recommendedPrice × ブランド係数
```

| ブランド設定 | 係数 |
|------------|------|
| 控えめ | 1.05 |
| 標準 | 1.15 |
| 高め | 1.30 |

### 8. 価格の丸め方

| モード | 動作 |
|--------|------|
| 1円単位 | そのまま |
| 10円単位 | 10円単位に切り上げ |
| 100円単位 | 100円単位に切り上げ |
| 端数調整あり | 心理価格帯（1000・1200・1500・1800・2000・2500・2800・3000…）の中から最適な価格を選択 |

## 起動方法（ローカル開発）

### 必要なもの

- Node.js 18以上
- npm

### インストールと起動

```bash
# リポジトリをクローン
git clone https://github.com/kitakita-lab/accessory-price-simulator.git
cd accessory-price-simulator

# 依存パッケージをインストール
npm install

# 開発サーバーを起動
npm run dev
```

ブラウザで `http://localhost:5173` を開いてください。

## GitHub Pages への公開方法

### 方法1：手動デプロイ（gh-pagesパッケージを使用）

```bash
# gh-pages パッケージをインストール
npm install --save-dev gh-pages

# package.json の scripts に追加
# "deploy": "gh-pages -d dist"

# ビルドしてデプロイ
npm run build
npm run deploy
```

### 方法2：GitHub Actions（自動デプロイ）

`.github/workflows/deploy.yml` を作成：

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm install
      - run: npm run build
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
```

GitHubリポジトリの Settings → Pages → Source を `gh-pages` ブランチに設定してください。

## 今後の改善案

### 機能追加

- **EC販売向けモード**：送料・決済手数料・プラットフォーム手数料を加味した価格計算
- **委託販売向けモード**：委託手数料（30〜50%）を反映した価格計算
- **商品カテゴリ別テンプレート**：ピアス・イヤリング・ネックレス・ブレスレットごとの初期値設定
- **複数商品の一覧比較**：複数商品の価格・利益率を並べて比較
- **値札POPの出力**：計算結果をそのまま値札やポップ用に印刷できる機能
- **利益シミュレーション一覧表**：販売数×単価のマトリクスで利益を一覧表示
- **イベント全体の売上予測**：全商品の合計売上・合計利益・回収率をシミュレーション

### UX改善

- ダークモード対応
- 入力値のCSVエクスポート
- 過去の計算履歴の保存
- 共有URL機能（入力値をURLパラメータで共有）
- PWA対応（ホーム画面に追加してオフラインでも使えるように）
